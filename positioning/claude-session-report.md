# Отчёт сессии: позиционирование rsspace

Контекст для будущих чатов. Не рабочий документ — история решений и их причины. Финальные тексты: `positioning.md`. Продуктовые механики: `product-notes.md`.

## Что было сделано

Ревью и пересборка позиционирования от чернового брифа до финального документа из 8 разделов. Все разделы прошли построчное утверждение в режиме вопрос-ответ.

## Ключевые решения и ПОЧЕМУ

**Формат «тейк → боль → контраст».** Каждый блок п. 3: короткое утверждение, затем абзац с болью рынка и нашей позицией. Причина: документ из одних тейков — брошюра, по которой нельзя работать; механика реализации при этом выносится в product-notes, чтобы позиционирование не устаревало со скоростью продукта.

**Foodservice ERP как самоопределение** (п. 1). Термин взят из отдельно подготовленной термин-карточки (см. wiki anvx-base). Граница рынка — по операционной модели (есть кухня), не по типу заведения. Розница без кухни — горизонт v2, в документе не упоминается (лежит в positioning-ext-retail.md).

**«Коридорный подход» сужен.** Изначально словом называлось всё подряд (обучение, забота, «ведём по пути»). Сведён к реальному термину из корпоративного онбординга: обучение встроено в работу, не отрывает от смены. Инструменты: quick guide, шаблоны техкарт, пошаговые сценарии, FAQ, специалист как куратор.

**Курс ремесла — отдельный продукт.** Грин поймал риск: «учим ремеслу» звучало как часть подписки 2500₽, но полноценное обучение — платная услуга. Решение: freemium (бесплатный слой в комплекте + опросник в интерфейсе + платный курс), в позиционировании формулировки не разделяли, но риск зафиксирован в product-notes.

**Поддержка.** Заказчик в опросе выбрал «специалист, не бот», но буквальная трактовка нежизнеспособна (изнеможение на 15 клиентах). Решение по карт-бланшу: бот-триаж как короткое уточнение перед человеком — не противоречит духу выбора (отвергалась бот-стена). Три канала по срочности: телефон (мгновенно) → чат (спокойно) → имейл (техпроблемы). Позиция в документе: «человек, не бот + в ритме ресторана».

**Три фронта в отличиях** (п. 4) — по April Dunford (competitive alternatives): iiko/r_keeper (уходящие: без посредников, помощь в переносе, счёт не растёт) · Quick Resto/Saby (сделано для себя + коридор) · «зоопарк» Excel/тетрадь/голова (скрытый счёт бардака + низкий порог входа). Причина третьего фронта: огромная часть рынка не выбирает между системами вообще.

**«Помогаем перенести», не «переносим автоматически».** LLM-конверсия данных не тестировалась. План теста в product-notes (критерий: тихие ошибки фатальны, явные — нет).

**Раздел «От чего отказываемся»** (п. 5) — по Портеру (trade-offs защищают позицию): удержание замком, экономия на поддержке, кастом, модульная монетизация. Порядок задан Грином.

**Тон бренда** (п. 8) — по методике NN/g: не эпитеты, а 2 черты с примерами «это мы / это не мы». «Прямой» и «наставнический» отклонены (черты людей, не компании). Осталось: тёплый/человечный + «верим в то, что делаем» (референс — Apple: репутация отношением, не рекламой).

## Отклонённые идеи (не поднимать заново без причины)

- Момент входа клиента — это для рекламщика, не для позиционирования
- Ограничение по масштабу («не для 50+ точек») — режет аудиторию, противоречит п. 1
- Упоминание розницы/бизнеса без кухни в документе — очевидно из определения рынка
- Черта «инсайдерский/клуб» — заменена на «верим в то, что делаем»
- Связка-мостик п. 8 к главной идее — тон остаётся отдельным слоем

## Открытые хвосты

- Тест переноса данных (план в product-notes) — нужна реальная выгрузка
- On-premise/коробка — не обсуждалось, не исключено
- SLA поддержки — вопрос будущего, публично не обещаем
- Есть ли у заказчика своя розница — спросить при случае (для v2)
