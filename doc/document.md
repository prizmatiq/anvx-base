# Дополнительный синтаксис разметки базы

Каждый док в этой "Базе" написан на .md (Markdown), но к сожалению, в обычном маркдауне нет всего, что нужно для хорошего справочного материала, поэтому были созданны дополнительные элементы.

Обычный markdown (заголовки, списки и т.д.) работает как всегда — здесь только то, что добавлено сверх стандарта.

## Как добавить новый документ

- Скопировать terms/folder, переименовать под новый термин — внутри ничего не менять
- Рядом положить .md-файл с точно именем document.md
- Добавить ссылку на новую страницу в terms/index.html [это пока вручную]

## Списки да / нет

```
:::do
- Foodservice ERP
- ERP-платформа для foodservice
:::

:::do-not
- программа для общепита
- система автоматизации ресторана
:::
```

Блоки :::do и :::do-not должны идти подряд через пустую строку — тогда получится двухколоночный блок с галочкой/крестиком. Если один из блоков без пары — получится просто список с иконкой, в одну колонку.

## Контакт Telegram и Email

```
:::contact @prizmatiq

:::contact hello@anvx.ru

:::contact https://t.me/joinchat/AbCdEfGh
```

Одна строка, без закрывающего `:::`. Первый вариант — юзернейм Telegram, второй — email (mailto), третий — прямая ссылка (для инвайтов в чаты без юзернейма).
